## ЁЯЦея╕П Overview

This is a **complete bspwm desktop environment** тАФ not just a window manager config, but a fully cohesive system covering every layer of the desktop: window management, hotkeys, bar, widgets, terminal, editor, launcher, notifications, right-click menu, GTK theming, shell, and fetch. Everything is color-coordinated and works together out of the box.

Built on **bspwm** тАФ the binary space partitioning window manager тАФ where every window placement decision is driven by a clean tree-based algorithm. Paired with **sxhkd** for hotkeys and an **eww dashboard** for custom widgets built in SCSS.

### At a glance

| Layer | Tool |
|---|---|
| Window Manager | bspwm |
| Hotkey Daemon | sxhkd |
| Terminal | Alacritty |
| Shell | Zsh (`.zshrc` included) |
| Editor | Neovim (Lua config) |
| Widget System | eww (SCSS dashboard) |
| App Launcher | Rofi |
| Notifications | Dunst |
| Right-click Menu | jgmenu |
| Audio Visualizer | Cava |
| GTK Theme | gtk-2.0 + gtk-3.0 |
| Fetch | Neofetch (custom) |
| Scripting | Shell + PowerShell |

## ЁЯзй Full Stack

### ЁЯкЯ bspwm тАФ Window Manager

bspwm represents all windows as leaves of a full binary tree. Unlike i3's manual tiling, bspwm is purely controlled via `bspc` commands тАФ there is no built-in keybinding system, which is why sxhkd exists as a separate daemon.

Key characteristics of this config:
- Binary space partitioning тАФ windows split the available space automatically
- External rule system тАФ window rules written as clean `bspc rule` commands
- Gap configuration тАФ inner/outer gaps for breathing room
- Multiple desktop (workspace) support
- Autostart тАФ launches sxhkd, eww, and compositor on startup

### тМия╕П sxhkd тАФ Hotkey Daemon

sxhkd (Simple X Hotkey Daemon) handles all keyboard shortcuts independently of bspwm. It reads an `sxhkdrc` file and executes shell commands on keypress. This separation means keybinds work even if the WM crashes.

### ЁЯЦея╕П Alacritty тАФ Terminal

GPU-accelerated terminal emulator written in Rust. This config includes:
- Custom color scheme matching the overall palette
- Font configuration with Nerd Font support
- Opacity / background blur settings
- Padding and scrollback tuning

### ЁЯРЪ Zsh тАФ Shell
`.zshrc` at repo root

A full Zsh configuration included at the root level:
- Custom prompt (likely Starship or manual PS1)
- Aliases and functions
- Plugin configuration
- Consistent with the terminal color theme

### ЁЯУЭ Neovim тАФ Editor
`nvim/` тАФ **21.7% of the repo is Lua**

A complete Neovim config written entirely in Lua тАФ the modern standard for Neovim configuration. Includes:
- Plugin manager setup (likely lazy.nvim or packer)
- LSP configuration for code intelligence
- Syntax highlighting via Treesitter
- Custom colorscheme matching the desktop theme
- Statusline, file explorer, fuzzy finder

### ЁЯкЯ eww тАФ Widget Dashboard
`eww/dashboard/` тАФ **14.5% of the repo is SCSS**

Elkowar's Wacky Widgets тАФ a fully custom widget system written in Yuck (markup) and styled with SCSS. This is the most technically sophisticated component in the setup:
- Custom dashboard panel (system stats, clock, widgets)
- SCSS styling for pixel-perfect design control
- Reactive widgets that update from shell scripts
- Styled to match the overall color theme

This is what separates a basic rice from a proper desktop environment.

### ЁЯЪА Rofi тАФ App Launcher
`rofi/`

Themed application launcher with:
- Custom `.rasi` stylesheet
- dmenu mode for scripts
- Window switcher integration with bspwm desktops
- Color scheme matching the full palette

### ЁЯФФ Dunst тАФ Notification Daemon
`dunst/`

Lightweight notification daemon replacing the default system notifications:
- Custom `dunstrc` with themed colors
- Position, timeout, and size configuration
- Icon support
- Urgency-based color differentiation (low / normal / critical)

### ЁЯУЛ jgmenu тАФ Right-click Desktop Menu
`jgmenu/`

A right-click context menu for the desktop тАФ uncommon in modern rices, making this setup more complete:
- Custom theme matching the desktop palette
- Configured menu items and categories
- Clean minimal styling

### ЁЯОи GTK Theming тАФ gtk-2.0 & gtk-3.0
`gtk-2.0/` and `gtk-3.0/`

Full GTK application theming тАФ meaning GUI apps like file managers, browsers, and dialogs match the desktop aesthetic:
- `gtk-2.0/` тАФ legacy GTK2 app support
- `gtk-3.0/` тАФ modern GTK3 app theming
- Consistent colors with the rest of the rice
- Custom cursor and icon theme references

### ЁЯО╡ Cava тАФ Audio Visualizer
`cava/`

Terminal-based audio spectrum visualizer configured to match the color theme. Can run in a dedicated Alacritty window as a visual accent.

### ЁЯУЯ Neofetch тАФ System Fetch
`neofetch/`

Custom neofetch configuration with:
- Custom ASCII art or image display
- Selected info fields
- Color output matching the palette

### ЁЯТ╗ PowerShell
`powershell/` тАФ **13.8% of the repo**

PowerShell scripts included for cross-platform utility or Windows-side dotfile management тАФ rare in a Linux rice and shows a multi-platform workflow.

## тЪЩя╕П Installation

### Prerequisites

| Requirement | Notes |
|---|---|
| Arch Linux / BlackArch | Or any Arch-based distro |
| Xorg | bspwm requires X11 |
| AUR helper | `yay` or `paru` recommended |
| Nerd Fonts | For icons in bar, terminal, nvim |

### Step 1 тАФ Clone

```bash
git clone https://github.com/0xhroot/bspc-dotfile.git
cd bspc-dotfile
```

### Step 2 тАФ Install core dependencies

```bash
sudo pacman -S bspwm sxhkd alacritty rofi dunst picom feh zsh neovim \
               cava neofetch jgmenu

# AUR packages
yay -S eww-git
```

### Step 3 тАФ Copy configs

```bash
# Create config directories
mkdir -p ~/.config/{bspwm,sxhkd,alacritty,rofi,dunst,eww,nvim,cava,jgmenu,neofetch}

# Copy each component
cp -r alacritty/*   ~/.config/alacritty/
cp -r cava/*        ~/.config/cava/
cp -r dunst/*       ~/.config/dunst/
cp -r eww/*         ~/.config/eww/
cp -r gtk-2.0/*     ~/.config/gtk-2.0/
cp -r gtk-3.0/*     ~/.config/gtk-3.0/
cp -r jgmenu/*      ~/.config/jgmenu/
cp -r neofetch/*    ~/.config/neofetch/
cp -r nvim/*        ~/.config/nvim/
cp -r rofi/*        ~/.config/rofi/
cp    .zshrc        ~/.zshrc

# Make bspwm scripts executable
chmod +x ~/.config/bspwm/bspwmrc
chmod +x ~/.config/sxhkd/sxhkdrc
```

### Step 4 тАФ Set Zsh as default shell

```bash
chsh -s $(which zsh)
```

### Step 5 тАФ Install Neovim plugins

```bash
nvim  # Open Neovim тАФ plugins will auto-install on first launch
```

### Step 6 тАФ Restart or login to bspwm

If already in a session:
```bash
bspc wm -r
```

Or log out and select bspwm from your display manager.

## тМия╕П Keybindings

> `super` = Win key. All binds are defined in `sxhkd/sxhkdrc`.

### Window Management

| Keybind | Action |
|---|---|
| `super + Return` | Open terminal (Alacritty) |
| `super + d` | Open Rofi launcher |
| `super + q` | Close focused window |
| `super + f` | Toggle fullscreen |
| `super + space` | Toggle floating |
| `super + t` | Set tiling mode |

### Focus & Move

| Keybind | Action |
|---|---|
| `super + h/j/k/l` | Focus window (left/down/up/right) |
| `super + shift + h/j/k/l` | Move window in direction |
| `super + alt + h/j/k/l` | Resize window |

### Desktops (Workspaces)

| Keybind | Action |
|---|---|
| `super + 1тАУ9` | Switch to desktop |
| `super + shift + 1тАУ9` | Send window to desktop |
| `super + Tab` | Last desktop |

### System

| Keybind | Action |
|---|---|
| `super + shift + r` | Reload sxhkd config |
| `super + shift + q` | Quit bspwm |
| `super + e` | Open eww dashboard |

> Edit `sxhkd/sxhkdrc` to see and customize all binds.

## ЁЯЖЪ bspwm vs i3 тАФ Why the switch?

This repo is a natural evolution from the [i3-dotfile](https://github.com/0xhroot/i3-dotfile). Here's what changed:

| Feature | i3wm | bspwm |
|---|---|---|
| Layout algorithm | Manual tiling | Binary space partitioning |
| Keybind system | Built-in | External (sxhkd) тАФ more flexible |
| Scriptability | Limited | Full тАФ every action is a `bspc` CLI call |
| Widget system | Polybar only | eww тАФ full custom SCSS widgets |
| Config language | DSL | Shell scripts + sxhkdrc |
| Learning curve | Lower | Higher тАФ more powerful |
| GTK theming | тЭМ not in i3 rice | тЬЕ full gtk-2.0 + gtk-3.0 |
| Notifications | тЭМ not in i3 rice | тЬЕ Dunst |
| Editor config | тЭМ not included | тЬЕ Full Neovim Lua config |
